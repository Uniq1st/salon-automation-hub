# Contributing Guidelines

Thank you for your interest in contributing! Here's how to get started.

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others grow

## Development Setup

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/salon-automation-hub.git
   cd salon-automation-hub
   ```
3. **Create a branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Install dependencies**:
   ```bash
   npm install
   ```
5. **Start development**:
   ```bash
   npm run dev
   ```

## Making Changes

### Code Style
- Use 2-space indentation
- Follow ESLint rules: `npm run lint`
- Add JSDoc comments for functions

### Testing
```bash
# Before committing
npm test                 # Run tests
npm run test:coverage    # Check coverage
npm run lint             # Lint code
```

### Commit Messages
Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add customer segmentation feature
fix: correct AI message generation timeout
docs: update deployment guide
test: add integration tests
```

## Submitting Changes

1. **Push** to your fork
2. **Create a Pull Request** on GitHub
3. **Describe** what changed and why
4. **Wait** for review and feedback
5. **Make** requested changes
6. **Celebrate** when merged! 🎉

## PR Requirements

- ✅ Tests pass
- ✅ Code is linted
- ✅ Documentation updated
- ✅ No sensitive data exposed
- ✅ Clear commit messages

## Reporting Bugs

1. Check [existing issues](https://github.com/yourusername/salon-automation-hub/issues)
2. Create [new issue](https://github.com/yourusername/salon-automation-hub/issues/new)
3. Include:
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Environment (OS, Node version, etc.)

## Feature Requests

1. Check [discussions](https://github.com/yourusername/salon-automation-hub/discussions)
2. Describe the use case
3. Explain why it's valuable
4. Suggest implementation approach
